#### 2. Постановка задачи
Необходимо реализовать консольное приложение на **C++ (C++11 и выше)**, моделирующее обработку задач несколькими потоками.
Основные требования:
- Реализация потокобезопасной очереди `TaskQueue`
- Использование `std::mutex` и `std::condition_variable` для синхронизации
- Исключение состояния busy-wait
- Корректное завершение всех потоков после выполнения 20 задач

##### Архитектура программы
Программа реализует многопоточную обработку задач с использованием стандартной библиотеки C++11.

Основными компонентами программы являются:
- класс `TaskQueue` - потокобезопасная очередь задач
- рабочие потоки (`worker threads`)
- главный поток (`main`)

Главный поток создает очередь задач, запускает рабочие потоки и добавляет в очередь задачи с числами от 1 до 20. После добавления всех задач вызывается завершение работы очереди и выполняется ожидание завершения всех потоков через `join()`.

Рабочие потоки параллельно получают задачи из общей очереди, обрабатывают их в течение одной секунды и выводят информацию о выполнении задачи в консоль.

##### Потокобезопасная очередь
Для хранения задач используется контейнер `std::queue<int> tasks;` . Так как очередь используется несколькими потоками одновременно, доступ к ней защищен мьютексом: `std::mutex mtx;` .  Для блокировки мьютекса применяется: `std::unique_lock<std::mutex>` .

Метод `push()`:
- блокирует доступ к очереди
- добавляет задачу
- уведомляет один ожидающий поток через `notify_one()`

Метод `pop()`:
- ожидает появления задач
- извлекает задачу из очереди
- возвращает `false`, если работа очереди завершена и задач больше нет

Благодаря использованию мьютекса исключаются гонки данных при работе с общей очередью.

##### Использование `condition_variable`
Для синхронизации потоков используется `std::condition_variable cv;` .
Если очередь задач пуста, поток переходит в состояние ожидания: `cv.wait(lock, [this] { return !tasks.empty() || stop_flag; });` .

Использование `condition_variable` позволяет:
- избежать busy-wait
- не выполнять постоянную проверку очереди в цикле
- эффективно использовать ресурсы процессора

После добавления новой задачи вызывается `cv.notify_one();` , что пробуждает один ожидающий поток.

При завершении программы используется `cv.notify_all();` , это необходимо для пробуждения всех потоков, ожидающих задачи, чтобы они могли корректно завершить выполнение.

##### Механизм корректного завершения
Для завершения работы потоков используется логический флаг `bool stop_flag = false;` .

После добавления всех задач главный поток вызывает метод `shutdown()`, который:
- устанавливает `stop_flag = true`
- уведомляет все ожидающие потоки через `notify_all()`

После пробуждения поток проверяет:
- есть ли задачи в очереди
- установлен ли флаг завершения

Если очередь пуста и установлен `stop_flag`, метод `pop()` возвращает `false`, из-за чего завершается цикл обработки задач: `while (queue.pop(task))` .

После завершения работы главный поток вызывает `t.join();` , это позволяет дождаться завершения всех рабочих потоков и корректно завершить программу без зависаний и незавершенных потоков.
